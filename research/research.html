Продукт

МемКат

Відеоредактор для блогерів із вбудованою бібліотекою мемів

⚡
Проблема

Щоб зробити мем-відео, доводиться перемикатись між редактором, Giphy, TikTok і YouTube — шукаєш меми, звуки, тренди по різних застосунках. Рутина вбиває темп.

🎯
Рішення

Усе в одному вікні. Зняв → додав мем → виклав. Нуль перемикань між застосунками. Вбудована бібліотека мемів (Giphy, Tenor) прямо в редакторі.

👥
Аудиторія

Контент-мейкери TikTok / YouTube Shorts / Reels, 16–30 років. Монтують швидко й часто, цінують актуальні тренди, не хочуть витрачати час на рутину.

📱
Платформа

Веб, mobile-first. Адаптив на десктоп і планшет. Імпорт відео з будь-якого пристрою — телефон, ПК, ноут.

🚀
MVP

Імпорт відео · Таймлайн (обрізка, розрізання, порядок) · Накладання мема / стікера · Додавання звуку · Текст вручну · Експорт 9:16

💸
Інфраструктура

$0. WebCodecs (рендер на пристрої) + IndexedDB (локальне сховище) + Giphy/Tenor API (безкоштовний tier) + Cloudflare Pages / Vercel. Реальні витрати — домен ~$10–15/рік.

Позиціонування Не «ще один редактор» — найшвидший спосіб зробити мем-відео. Усе що не працює на цю тезу — ріжемо.

Топ-3 механізми для MVP

Механізм 1 ← Telegram 40/40
Емоджі як пошуковий запит

Юзер вводить 😂 і отримує релевантні GIF — природно, без слів. Giphy і Tenor API підтримують emoji search нативно. Одне поле вводу — нульова додаткова інфраструктура.

developer.giphy.com · tenor.com/gifapi
Механізм 2 ← Telegram + Discord
Нещодавні вгорі + trending без запиту

Пікер відкривається — одразу корисний контент. Нещодавні → IndexedDB ($0). Trending → один Giphy/Tenor запит при відкритті.

benchmark · CLAUDE.md (IndexedDB)
Механізм 3 ← Telegram + Gboard
Inline panel, не full-screen overlay

Пікер знизу як панель, таймлайн залишається видним. Discord і Slack (overlay, 3/5) підтверджують: overlay збиває фокус і сповільнює.

benchmark
Не спрацює
Локальне кешування всієї бібліотеки (Telegram-стиль)

Telegram кешує пакети (~20–100 стікерів, фіксований обсяг — юзер сам обирає). У МемКат: Giphy/Tenor — 10M+ GIFs, динамічна, нескінченна. Кешувати все неможливо в IndexedDB при $0 інфраструктури.

Рішення: кешувати тільки нещодавно переглянуті (~50 GIF в IndexedDB). Решта — lazy load з CDN зі skeleton placeholders.
03

Патерни

З 5 принципово різних патернів (Zero-query browse / Instant text search / Emoji-as-intent / Curated category tabs / Recents + Favorites) обрано основний і fallback.

Обраний патерн
Emoji-as-intent

Юзер вводить емоджі замість тексту — отримує контент що відповідає настрою. Emotional shorthand. Єдиний масовий продукт з цим патерном як ядром — Telegram (40/40 у бенчмарку).

  • 1
    Аудиторія 16–30 думає настроями, не словами. Intent мем-мейкера — "хочу щось типу 😂 але тупо", а не "funny reaction GIF". Текстовий пошук вимагає сформулювати слово, емоджі — ні. [benchmark: Telegram 40/40]
  • 2
    Giphy і Tenor API підтримують emoji query нативно — $0 додаткової інфраструктури. Надсилаєш емоджі як query параметр — отримуєш GIF. Вкладається в обмеження $0 з CLAUDE.md. ? Потребує верифікації в API docs перед реалізацією
  • 3
    Різить від усіх конкурентів. Жоден відеоредактор не має emoji search — підтверджено при ресерчі. МемКат стає першим. Підтримує тезу "найшвидший спосіб зробити мем-відео". [WebFetch + скріни capcut, kapwing, veed, clideo, canva]
Fallback — за умови cold start
Zero-query browse (Trending-first)

Новий юзер — history порожня, не знає про emoji search. Trending endpoint дає актуальний контент без дій. Один API-запит при відкритті — $0. Onboarding і fallback одночасно.

Не підходить
Curated category tabs

Меми живуть тиждень — категорії застарівають. Підтримка = контент-команда або ML. У МемКат: один розробник, $0, без власної бази. Trending endpoint Giphy/Tenor виконує цю роботу краще і безкоштовно.

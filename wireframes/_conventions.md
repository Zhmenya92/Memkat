# Wireframe Conventions — МемКат

*28.06.2026 · Діє на всі wireframes проекту*

---

## 1. Деталізація

Wireframe показує лише структуру — нічого більше:

**Що включаємо:**
- Ієрархію зон і контейнерів
- Розміщення і відносні пропорції елементів
- Навігаційні точки та переходи між зонами
- Стани елементів (disabled / active / placeholder)

**Що не включаємо:**
- Кольори — тільки відтінки сірого
- Іконки у фінальному вигляді — прямокутник або коло-заглушка
- Реальні зображення або відео — сірий прямокутник з діагональним хрестом
- Анімацію та мікровзаємодію

## 2. Темна палітра

Wireframes використовують **темну тему** — відтінки темно-сірого по шкалі від `#0A0A0A` до `#EDEDED`.

Префікс токенів — **`--`** (без `wf-`). Так у всіх 21 файлі; раніше тут було `--wf-`, але реалізація одностайна на `--`, тож еталон оновлено під неї.

```css
/* Базові (в кожному файлі) */
--bg:               #141414   /* фон сторінки */
--surface:          #1E1E1E   /* картки, sheet-и, підняті контейнери */
--zone:             #272727   /* зони, вторинні блоки */
--zone-deep:        #2F2F2F   /* таймлайн, toolbar */
--border:           #383838   /* рамки, divider */
--text:             #EDEDED   /* основний текст */
--text-2:           #8A8A8A   /* вторинний текст, мітки */
--text-3:           #515151   /* placeholder, підказки */
--interactive-pri:  #D8D8D8   /* primary CTA — світлий fill на темному тлі */
--interactive-pri-fg: #141414 /* текст primary CTA */

/* Контекстні (тільки де потрібні) */
--scrim:  rgba(0,0,0,.65)        /* scrim за bottom sheet — picker, share */
--err:    #C05050                /* error-стани */
--err-bg: rgba(192,80,80,.08)    /* фон error-блоку */
--col:    480px                  /* колонка контенту на планшеті (640–1023) — import, export */
```

Фон за межами телефону (body): `#0A0A0A`.

**Прибрано:** `--interactive` (#B0B0B0) і `--interactive-bg` (#2A2A2A) — у жодному файлі не використовуються. Secondary-кнопки малюються через `transparent + 1px border var(--border)`, не через окремий fill-токен.

## 3. Розмітка

Тільки семантичний HTML — жодних безіменних `<div>` там де є семантичний аналог.

| Елемент | Де використовувати |
|---|---|
| `<header>` | Шапка екрана: назва + contextual actions |
| `<nav>` | Tab bar, breadcrumb, будь-яка навігаційна зона |
| `<main>` | Основний контент екрана |
| `<section>` | Логічна зона всередині `<main>` |
| `<article>` | Повторювана самостійна одиниця (картка проекту, GIF-елемент) |
| `<aside>` | Другорядна зона (sidebar, info panel) |
| `<form>` | Будь-яка форма або пошукове поле |
| `<button>` | Будь-яка активна кнопка — не `<div>`, не `<a>` без href |
| `<label>` | Підпис до поля введення |
| `<dialog>` | Modal або bottom sheet |
| `<progress>` | Progress bar (рендер, завантаження) |

`<div>` і `<span>` — тільки для суто верстальних обгорток без семантичного сенсу.

## 4. Текст

Справжній текст з домену МемКат — ніякого lorem ipsum.

**Назви проектів:** «Відео для TikTok · 28 черв», «Мем з котом · 26 черв», «Перший ролик · 14 черв»

**Кнопки:** «Новий проект», «Додати мем», «Імпортувати відео», «Експортувати», «Поділитись», «Спробувати інший файл», «Скасувати»

**Пошук — підказки:** 😂 funny · wow · omg · fail · relatable

**Повідомлення помилок:**
- «Файл завеликий — максимум 500 МБ»
- «Формат не підтримується — спробуйте MP4 або MOV»
- «Не вдалось зберегти · Спробувати ще»
- «Не вдалось завантажити GIF · Повторити»
- «Помилка рендеру · Повторити або повернутись до редактора»

**Toast:** «Збережено», «GIF додано», «Формат змінено»

**Форматна підказка:** «9:16 — TikTok / Reels / Shorts», «1:1 — Instagram feed»

## 5. Назви файлів

Базова сторінка = **happy path / success state** — без суфікса.
Кожен додатковий стан — окремий файл із суфіксом.

```
wireframes/<назва>.html           ← базова (success / happy path)
wireframes/<назва>-empty.html     ← порожній стан
wireframes/<назва>-error.html     ← стан помилки
wireframes/<назва>-loading.html   ← стан завантаження
```

**Повна таблиця файлів:**

| Тип | Одиниця | base | -empty | -error | -loading |
|:---:|---|---|---|---|---|
| 📱 | Список проектів | `project-list.html` | `project-list-empty.html` | `project-list-error.html` | — |
| 📱 | Вибір відео | `import.html` | — | `import-error.html` | `import-loading.html` |
| 📱 | Редактор | `editor.html` | `editor-empty.html` | `editor-error.html` | `editor-loading.html` |
| ⧉ | Мем-пікер | `picker.html` | `picker-search-empty.html` | `picker-error.html` | `picker-loading.html` |
| 📱 | Екран експорту | `export.html` | — | `export-error.html` | `export-loading.html` |
| ⧉ | Поділитись | `share.html` | — | — | — |

**Разом: 14 файлів** — відповідає таблиці станів у `_screens.md`.

**Примітки:**
- `picker.html` (base) = пікер відкритий з контентом (Trending tab, cold start)
- `picker-search-empty.html` = вкладка Пошуку до введення запиту
- `share.html` = єдиний стан компонента, він і є «вийшло»

## 6. Стани — деталі кожного суфікса

**base (без суфікса):**
Екран або компонент у нормальному робочому стані. Всі елементи активні, є контент.

**-empty:**
Порожній стан — ще немає контенту. Обов'язково: ілюстрація або іконка-заглушка + заголовок + один головний CTA.
Приклад: Список проектів без жодного проекту → «Почніть перший проект» + кнопка «Новий проект».

**-error:**
Щось пішло не так. Структура: опис проблеми одним реченням + CTA «Спробувати ще» або альтернативна дія.
Приклад: `import-error.html` — блок над файл-зоною з текстом помилки + двома кнопками.

**-loading:**
Процес виконується. Обов'язково: індикатор (`<progress>` або spinner-заглушка) + текст що саме відбувається.
Приклад: `export-loading.html` — progress bar з підписом «Рендер відео...» + кнопка «Скасувати».

## 7. Візуальний стиль

Wireframe має читатись як дизайн-документ, а не технічний скетч.

**Сітка:**
- Базова одиниця: 8px
- Відступи: 8 / 16 / 24 / 32 / 48px
- Mobile viewport: 390px × 844px (iPhone 14 Pro reference)

**Типографіка (system-ui, sans-serif):**
| Роль | Розмір | Вага |
|---|---|---|
| Вордмарк «МемКат» (listings) | 30px моб / 22px 640+ | 700 |
| Заголовок екрана | 17px | 600 |
| Назва секції | 15px | 600 |
| Основний текст | 15px | 400 |
| Вторинний текст | 13px | 400 |
| Мітка / caption | 11px | 500 |

> Вордмарк — це логотип бренду на головному екрані (listings), а **не** «заголовок екрана». Він свідомо більший за 17px і не підпадає під правило заголовків. Решта екранів заголовка-лого не мають — у них `header-title` 17/600.

**Форма:**
| Елемент | border-radius |
|---|---|
| Кнопки маленькі | 8px |
| Кнопки великі / CTA | 12px |
| Картки проектів | 12px |
| Bottom sheet | 16px (верхні кути) |
| Input поля | 10px |
| Таблетка-мітка | 100px |

**Elevation (тільки для реальної глибини):**
- Bottom sheet: `box-shadow: 0 -4px 24px rgba(0,0,0,.12)`
- Картка: `box-shadow: 0 1px 4px rgba(0,0,0,.08)`
- Toast: `box-shadow: 0 4px 16px rgba(0,0,0,.14)`

**Placeholder медіа:**
- Відео / зображення: сірий прямокутник `background: #E4E4E4` + двi діагональні лінії `<svg>`
- Аватар / іконка: коло `background: #D8D8D8`
- GIF-карточка: прямокутник 1:1 або 4:3, `background: #E4E4E4`

**Інтерактивні елементи:**
- Primary CTA: `background: #D8D8D8; color: #141414; border-radius: 12px`
- Secondary: `background: #2A2A2A; color: #EDEDED`
- Disabled: `background: #202020; color: #404040; cursor: not-allowed`
- Destructive: `background: #2A2A2A; color: #C05050` (приглушений червоний текст — єдиний відступ від сірого)

**Скролбари:**
Тонкі, Figma-style — обов'язково в кожному wireframe одразу після `*, *::before, *::after`:

```css
::-webkit-scrollbar { width: 5px; height: 5px; }
::-webkit-scrollbar-track { background: transparent; }
::-webkit-scrollbar-thumb { background: rgba(255,255,255,.14); border-radius: 10px; transition: background .15s; }
::-webkit-scrollbar-thumb:hover { background: rgba(255,255,255,.30); }
::-webkit-scrollbar-corner { background: transparent; }
* { scrollbar-width: thin; scrollbar-color: rgba(255,255,255,.14) transparent; }
```

**Анотації (опціонально):**
Якщо wireframe потребує пояснень — додати `<aside class="wf-note">` праворуч або нижче. Не використовувати callouts всередині зон.

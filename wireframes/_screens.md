---

## Таксономія

| Тип | Позначення | Визначення |
|---|---|---|
| Екран | 📱 | Окрема точка навігації; push у navigation stack; Back повертає на попередній екран |
| Компонент | ⧉ | Overlay / bottom sheet поверх активного екрана; не навігаційна точка; екран за ним залишається активним |
| Стан | ⬜ 🔴 ⏳ ✅ | Empty / error / loading / success — стани всередині екрана або компонента |

---

## Відбір екранів та компонентів

**Критерій:** одиниця входить у happy path «побачила мем у стрічці → зняв → додав мем → виклав» (Path A, flows.md) + має підтверджений job з jtbd.md.

**Виключені:**
- Мем-пікер · Trending — не primary шлях Ані (R2, secondary персона Ліза)
- Мем-пікер · Нещодавні — Path B (повторний проект), не Path A
- Мем-пікер · Пошук emoji/mood — стан всередині компонента, не окрема одиниця
- Налаштування, Атрибуція — SIROTA, жодного job

## Екрани та стани

### 1. Список проектів

**Job (jtbd.md · Main):**
«Коли я бачу мем або тренд у стрічці, я хочу швидко зробити і викласти своє відео з ним, щоб потрапити в момент поки він живий.»

**Місце у flow (flows.md · Main):**
Крок 0 — вхідна точка петлі. `T ==> PL["📱 Список проектів"]`. CTA «+» → IMP (новий проект) або tap картки → ED (продовжити).

**Стани:**
- ⬜ Порожній — ✓. `history = 0`; стан явно описаний у sitemap і flows: «порожній стан з CTA «Новий проект»». Проектується окремо.
- 🔴 Помилка — ✓. Blob зник з IndexedDB (кеш очищено): «Відеофайл не знайдено» + CTA «Імпортувати знову». Документовано в sitemap.
- ⏳ Завантаження — —. Blob-перевірка IndexedDB прозора для юзера (явний коментар у sitemap: «прозора для юзера»). Видимого loading state немає.
- ✅ Успіх — —. Екран — точка входу, не завершення жодного job. Успіх не показується тут.

### 2. Вибір відео (імпорт)

**Job (jtbd.md · Main):**
«Коли я бачу мем або тренд у стрічці, я хочу швидко зробити і викласти своє відео з ним, щоб потрапити в момент поки він живий.» — крок «зняв»: відзнята сировина = перша ланка петлі.

**Місце у flow (flows.md · Main):**
Крок 1. `PL -. новий проект .-> IMP`. Після вибору файлу: `Q3{Файл OK?} == завантажено ==> ED`.

**Стани:**
- ⬜ Порожній — —. Файловий пікер пристрою = системний UI (не наш екран), sitemap: «системний UI — не наш екран». До вибору файлу наш екран не існує як окремий view.
- 🔴 Помилка — ✓. Три типи після вибору файлу: непідтримуваний формат / > 500 MB / файл пошкоджено. Дії: «Спробувати інший файл» або «Скасувати → 1». Документовано в sitemap і flows.
- ⏳ Завантаження — ✓. flows.md: «⏳ loading + 🔴 error — Вибір відео: завантаження → валідація» між вибором файлу і переходом до ED.
- ✅ Успіх — —. Успіх = перехід до Редактора. Окремого «файл прийнятий» екрана немає — flow одразу переходить до ED.

### 3. Редактор

**Job (jtbd.md · Main + R3):**
Main — «зняв → **додав мем** → виклав»: Редактор = основне середовище де відбуваються обидва кроки.
R3 — «Коли я монтую відео і хочу додати мем, я хочу знайти і вставити його не виходячи з монтажу, щоб не збирати фокус заново.»

**Місце у flow (flows.md · Main):**
Крок 2 — основний робочий простір. `Q3 == завантажено ==> ED`. Звідси: `ED == Додати мем ==> MP` і `ED == [Експорт] ==> EXP`.

**Стани:**
- ⬜ Порожній — ✓. R3 і R5 flows: `⬜ Редактор / Додати мем — disabled` і `⬜ Редактор / Експорт — disabled` коли таймлайн порожній. У Path A не трапляється (відео вже імпортоване), але стан реальний для будь-якого входу в редактор без кліпів.
- 🔴 Помилка — ✓. sitemap: «⚠ Autosave помилка — «Не вдалось зберегти» + retry». Також: помилка CDN при завантаженні GIF з пікера.
- ⏳ Завантаження — ✓. Autosave відбувається після кожної зміни; завантаження GIF з CDN після вибору у пікері — flows.md R3: «⏳ Завантаження GIF з CDN: спінер на Canvas поки буфер готовий».
- ✅ Успіх — ✓. sitemap: «✓ Autosave OK — silent toast «Збережено» (2 сек)». Видимий feedback що зміна збережена — окремий візуальний стан (toast), не загальний normal state.

### ⧉ Мем-пікер (bottom sheet) · Компонент

**Job (jtbd.md · R3):**
«Коли я монтую відео і хочу додати мем, я хочу знайти і вставити його не виходячи з монтажу, щоб не збирати фокус заново.»

**Місце у flow (flows.md · Main + R3):**
Крок 2а — inline picker всередині Редактора. `ED == Додати мем ==> MP["📱 Мем-пікер"]`. `MP == вибрала ==> EXP` (у Main flow пікер → вибір → далі до експорту); `MP -. закрила .-> ED`.

**Стани:**
- ⬜ Порожній — ✓ (вкладка Пошуку, не сам пікер). Сам пікер відкривається з контентом: cold start → Trending, history > 0 → Нещодавні. Порожній стан = лише вкладка Пошуку до першого введення: поле пусте, підказки 😂 funny wow.
- 🔴 Помилка — ✓. Два рівні: (1) API Giphy/Tenor недоступне — «🔴 Помилка API» + retry; (2) CDN помилка при завантаженні вибраного GIF — «🔴 Не вдалось завантажити» + retry + «вибрати інший». Обидва в sitemap і flows.
- ⏳ Завантаження — ✓. flows.md R1: «⏳ Завантаження · Giphy / Tenor» після введення запиту; sitemap: «⏳ Завантаження GIF — спінер після вибору GIF; CDN завантажує файл».
- ✅ Успіх — —. Успіх = GIF доданий на таймлайн → пікер закривається, повернення до Редактора. Ніякого «вийшло» всередині пікера немає — він просто зникає.

### 5. Екран експорту

**Job (jtbd.md · Main + R5):**
Main — третя ланка петлі «зняв → мем → **виклав**»: рендер і збереження файлу.
R5 — «Коли моє відео готове, я хочу зберегти його у правильному співвідношенні для платформи куди виходжу, щоб воно виглядало як треба без додаткових дій.»

**Місце у flow (flows.md · Main + R5):**
Крок 3. `MP == вибрала ==> EXP` (або `ED -- тапнула Експорт --> EXP`). Тут: вибір формату (9:16 / 1:1) → `EXP == підтвердила ==> REND["⏳ Рендер WebCodecs"]`.

**Стани:**
- ⬜ Порожній — —. Екран завжди відкривається з проектом (є кліп — інакше кнопка Експорт disabled). Немає визначеного empty state для самого екрана.
- 🔴 Помилка — ✓. flows.md Main + R5: `REND -. збій .-> REND_E["🔴 Помилка рендеру"]` + retry або «Повернутись до редактора». Реальний сценарій: WebCodecs збій або fallback рендер не проходить.
- ⏳ Завантаження — ✓. `⏳ Рендер WebCodecs` — явний стан з progress bar (flows.md R5: «видима смуга або % під час рендеру»). Також preview кадрування оновлюється live при виборі формату.
- ✅ Успіх — —. Успіх = перехід до Поділитись / завершити. Сам Екран Експорту переходу «вийшло» не показує.

### ⧉ Поділитись / завершити · Компонент

**Job (jtbd.md · Main + R5):**
Main — замикання петлі: «✅ Виклала / зняв → мем → виклав ✓».
R5 — файл збережений у правильному форматі; CTA share веде до системного share sheet платформи.

**Місце у flow (flows.md · Main):**
Крок 4 — фінальний екран. `REND == OK ==> SHARE["📤 Поділитись / завершити"]`. `SHARE == поділитись ==> OK(["✅ Виклала"])`.

**Стани:**
- ⬜ Порожній — —. Екран з'являється тільки після успішного рендеру — завжди є готовий файл.
- 🔴 Помилка — —. Помилки рендеру оброблені на попередньому екрані. Тут помилок немає (share sheet — системний UI).
- ⏳ Завантаження — —. Файл вже готовий до моменту появи екрана.
- ✅ Успіх — ✓. Цей екран і є «вийшло» — post-success state після рендеру. flows.md: `SHARE == поділитись ==> OK(["✅ Виклала\nзняв → мем → виклав ✓"])`. Весь екран = підтвердження виконаного Main job.
